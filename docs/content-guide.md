# Інструкція редактора

Увійдіть через Cloudflare Access. Створіть категорію, потім draft-публікацію: український title/body обов'язкові. Перевірте Markdown preview, alt/credit/license для фотографій, slug та SEO. Publish робить матеріал публічним. Для English заповніть окремі поля: не позначайте англійську версію published без title і body. Зміна slug повинна створити перевірений redirect.

## Видалення матеріалів

У поточній admin-політиці архівування через списки постів, сторінок і категорій не використовується: дія **«Видалити назавжди»** фізично видаляє запис, його pivot-зв'язки, redirects і створює audit event. Повне видалення доступне лише адміністратору. Категорію не можна видалити, якщо вона використовується постами або має дочірні категорії.

Для медіа видаляються D1 metadata та всі R2 variants. Медіа, пов'язане з постом, сторінкою або категорією, спочатку потрібно відв'язати; інакше сервер відхилить операцію. Видалення є незворотним, тому перед підтвердженням перевірте slug, зв'язки та резервну копію.
